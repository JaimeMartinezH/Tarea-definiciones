-Git Init:
Git Init es un comando que se usa para crear un nuevo repositorio de Git o re-inicializar uno existente. 
Con este comando, Git crea un directorio oculto llamado git que almacena toda la información y el historial de un 
proyecto.
Para mi, git init seria como una forma de empezar un nuevo proyecto con git o el de convertir un proyecto que ya existe
en un repositorio de github.
-Git config:
Es un comando que se usa para configurar las opciones de Git, como el nombre de usuario, el correo electrónico, el 
editor de texto, los alias, los colores y más.
Para mi es una forma de personalizar el aspecto y el funcionamiento de git según tus preferencias y necesidades.
-Git add:
Un comando que se usa para agregar el contenido de los archivos al indice de Git. Con este comando, Git selecciona los
archivos o partes de los archivos que se incluiran en el proximo commit.
Para mi es una forma de elegir que cambios quiero guardar con git.
-Git commit:
es un comando que se usa para guardar los cambios que se han agregado al índice de Git.
Para mi seria una forma de registrar el progreso de un proyecto con git y crear un punto de referencia para rehacer si 
es necesario.
-Git push:
Un comando que se usa para subir el contenido del repositorio local a un repositorio remoto.
Pra mi es una forma de compartir mis cambios locales con otros usuarios o en un servidor central.
-Git pull:
Un comando que se usa para actualizar el repositorio local con el contenido del repositorio remoto.
Para mi seria una forma de sincronizar tu proyecto local con los cambios de otros usuarios o de un servidor central.
-Git branch:
Es un comando que se usa para listar, crear o eliminar ramas de Git.
Para mi es una forma de trabajar en diferentes partes de un proyecto sin afectar al proyecto principal.
-Git checkout:
Es un comando que se usa para cambiar entre diferentes versiones de una entidad objetivo.
Para mi es una forma de explorar o restaurar diferentes estados de tu proyecto con git.
-Github:
Es una plataforma que ofrece un servicio de alojamiento de repositorios de Git en la nube. Básicamente, facilita el uso
de Git para el control de versiones y la colaboración entre desarrolladores.
Para mi seria un lugar donde puedes guardar, compartir y gestionar tu código con git.
-Github pages:
Es un servicio que ofrece GitHub para alojar páginas web estáticas a partir de archivos HTML, CSS y JavaScript que 
se encuentran en un repositorio de GitHub.
Para mi es una forma de crear y publicar una página web gratis usando github.
-Wiki:
Es una palabra que se refiere a un tipo de sitio web que permite a los usuarios editar el contenido de forma 
colaborativa usando un navegador web.
Para mi es una forma de crear y compartir información en línea entre muchas personas.
